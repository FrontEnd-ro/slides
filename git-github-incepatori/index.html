4) Pull requests

Atunci când funcționalitatea de pe un branch este complet implementată, facem un Pull Request (PR) pentru a o adăuga în master.

În cadrul unui PR ceilalți membri ai echipei dau feedback pe noul cod și sugestii de îmbunătățire

{ DEMO }

Task 2

1) Creați un PR către master

2) Hai să le merge-uim pe rând, fixând conflictele

git pull origin master
git add .
git commit -m "Fixed conflicts"
git push

Hai să aruncăm un ochi pe repo. Ar trebui să vedem codul tuturor pe repo 👀

5) Reverting

Cum revenim la o versiunea anterioară?

git revert {commit-sha}
git checkout {commit-sha}

Proiectul local va fi acum la acea versiune (nu are side-effects)

Adaugă un nou commit care face "undo" la toate schimbările
